PunktNameChanger Function Notes

This project renames Leica survey point IDs in `.imes`, `.ipkt`, `.iroh`, and
`.lqp` files while preserving file formatting and validating coordinates against
a selected master file.

Source of truth:

- Single-file mobile app: `Punkt-Name-Changer.html`
- Split implementation: `js/utils.js`, `js/parsers.js`, `js/renamer.js`, `js/main.js`
- Regression tests: `tests/run_validation.py`
- Security and deployment guidance: `SECURITY.md`
- Optional generated single-file build: `scripts/build_singlefile_dist.py`

Core behavior:

- Supported point IDs use dot format only: `G01.001..G10.998`,
  `P01.001..P10.998`, and `Q01.001..Q10.998`.
- `G` odd points receive suffix `03`; `G` even points receive suffix `04`.
- `P` odd points receive suffix `01`; `P` even points receive suffix `02`.
- `Q` groups map four source points to one MQ: positions 1/2 use suffixes
  `03`/`04`, positions 3/4 use suffixes `01`/`02`.
- Quadro prism positions, the third and fourth point in each `Q` group, add
  `0.04 m` to the existing height field during rename.
- Pattern mode starts at the configured source point and stops exactly at QTY.
- Coordinate checks must pass before a point is renamed.
- Replacement text is padded to preserve the original field width.
- Unsupported extensions are skipped before reading.
- Input files over 10 MB are skipped, and one selected session is capped at 30 MB total.
- Pattern base prefixes and output suffixes may contain only letters, numbers, dot, underscore, and hyphen.
- Files stay local in the browser session; no remote API, analytics, cookies, or browser storage are used.
- Content Security Policy metadata blocks network connections and object embedding.
- Busy status disables main action controls during file reads and rename runs.
- Export summary reports loaded files, modified files, and TXT log entries.
- The generated single-file build script writes only to `dist/` and does not modify `Punkt-Name-Changer.html`.

MQ numbering rule:

MQ is based on the original source point pair index:

```text
pairIndex = floor((sourceIndex - 1) / 2)
mqIndex = startMq + pairIndex - startPairIndex
```

This is intentional. If a field file contains only the first part of a path and
then jumps to the end of the path, the final section keeps its real MQ numbers
instead of being numbered as the next measured rows.

Example with start point `G01.001` and start MQ `1`:

```text
G01.001 -> MQ01
G01.016 -> MQ08
G01.071 -> MQ36
G01.088 -> MQ44
```
